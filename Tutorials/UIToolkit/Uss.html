    Introduction

    USS unity style sheet operate on the same Selector system as CSS by not as fleshed out. Like is CSS there is a hierarchy with goes * < Type < .Class < #Name < Inline-Style < Pseudo-Classes They way to think of it is the more specific the selector the more seniority it has.

    Selector

    • * Root Selector
      • This root selector matches to all elements and affect everything.
    • Type
      • The type of element such as “Button” this will affect all of the buttons.
      • Type overrides the * universal selector
    • .Class
      • The Class is represented with the “.ClassName” controls all elements with class name attached.
      • Class overrides Type

    • Name | #ID

      • The Name or (ID in CSS) is the name you give to an element(s) denoted by the “#Name” and any element with that name will be controlled by this.

      • Name overrides Class

    Pseudo Classes

    Pseudo classes like in CSS can be attached to Type, Class, Name and will affect if conditions are met.

    • :hover – when the mouse hovers over an element.
    • :active – When an Element is active (can be Interacted with).
    • :inactive – When an Element is inactive (can’t be Interacted with).
    • :focus – when an element is the current focus.
    • :selected – This is not implemented yet.
    • :disabled – When the element is Disabled.
    • :enabled – When the element is enabled.
    • :checked – When the element is toggled on.
    • :root – This is different from the other Pseudo classes as it is used by it’s self to define the default style for the element the style sheet affects

    You can combine multiple of these selectors together like in CSS.

    USS Data Types

    UI Toolkit style properties use the same grammar syntax as W3C CSS documents:
    Keyword values appear as is. For example: auto, baseline.
    Basic data types appear between angle brackets ( < and >). For example: , .
    Non-terminals that share the same name as a property appear between angle brackets and single straight quotes ( <' and '>). For example, <'width'>.
    If a property value has more than one component:
    Side-by-side words mean that all of them must occur in the given order.
    A bar (|) separates two or more alternatives: exactly one must occur.
    A double bar (||) separates two or more options: one or more must occur, in any order.
    A double ampersand (&&) separates two or more components, all of which must occur, in any order.
    Square brackets ([ ]) denote grouping.
    Every type, keyword, or angle-bracketed group may be followed by modifiers:
    an asterisk (*) indicates that the preceding type, word, or group occurs zero or more times.
    a plus (+) indicates that the preceding type, word, or group occurs one or more times.
    a question mark (?) indicates that the preceding type, word, or group is optional.
    a pair of numbers in curly braces ({A,B}) indicates that the preceding type, word, or group occurs at least A and at most B times.

    USS custom Properties (Variables)

    Variables like in CSS it uses the prefix of double-hyphen“--".
    For example “—color-1: red”;
    It can be used by var()
    For example var(--color-1);
    Variables is declared in a selector and can b e used by any child selectors, it is common practice to uses the variable in the root selector (*) as all other selectors can uses it.
