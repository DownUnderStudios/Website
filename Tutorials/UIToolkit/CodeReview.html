    Visual Element

    To be able to write code you need to understand how the code is written what can be used to make your UI
    I would always advocate using the UI Builder so you know how your UI looks at runtime, but there are some things that can done at runtime only, so you will need to understand the code you will be working with.
    We’ll start with the Visual Element as it is what all of the elements are derived from.

    Variables

    • Transform – is of type ITransform which contains the position, rotation, scale & matrix.
    • Visible – is whether the element is seen or not.
    • Enabled Self – if this element is enabled or not.
    • Enabled In Hierarchy – if this element is enabled or disabled by its Hierarchy.
    • Picking Mode – if this element can be picked during mouse events or IPanel.pick queries.
    • View Data Key – is data persistence such as scroll position & zoom level.
    • Usage Hints – can only be set when not part of a panel. Is used to boost performance.
      • None – not hint applicable.
      • Dynamic Transform – is for an element that change its transformation often.
      • Group Transform – is for an element that has many children that are Dynamic Transform.
      • Mask Container – is for an element that operates with “Overflow: hidden” style.
      • Dynamic Color – is for an element that changes any of its colors often.
    • Hierarchy – is a struct listed below.
    • Experimental – is for Transition Animations.
    • Resolved Style – style values after layout pass.
    • Tool Tips – when hovered over will display text informing what it is.
    • Style Sheet – are for the uss files attached to this element.
    • Style – is for the inline style of this element and overrides any other styles.

    Functions

    • Add – adds a child to this element.
    • Add To Class List – adds a class to this element.
    • Bring To Front will move this element to the end of the list of children in its parent.
    • Children – will return its children.
    • Class List Contains – returns true if it contains the class
    • Clear – removes all children from this element.
    • Clear Class List – removes all classes from this element.
    • Contains – returns true if this element is an ancestor of the child element.
    • Contains Point – if this element intersects with the point.
    • Element At – return the child at this index.
    • Enable In Class List – will add or remove the class based on the Boolean.
    • Find Common Ancestor – check closest ancestor that this and the other element share.
    • Focus - is a sealed function that was inherited from Focusable class and isn’t inheritable after visual element.
    • Get Classes – returns a list of the classes.
    • Index Of – returns and integer of the child or -1 if it isn’t found.
    • Insert – element at index.
    • Mark Dirty Repaint – when triggered will force the element to refresh on next frame.
    • Place Behind – will place this element behind its sibling.
    • Place In Front – will place this element in front of its sibling.
    • Remove – will remove this child.
    • Remove At – will remove child at index.
    • Remove From Hierarchy – will remove this element from its parent
    • Send Event – will send an event to the event handler, this function is also sealed and can’t be override from classes that inherit from this one.
    • Send To Back – is the opposite from Bring to Front as it will move this element to the top of its parents’ children list.
    • Set Enabled – same as Game object’s Set Active function, disabled elements don’t receive most events.
    • Toggle In Class List – if the class exist then remove it, if the class doesn’t exist then add it.

    Classes

    • UXML Factory – Instantiates this visual element from the data stored in the UXML file.
      • Inherits from UXMLFactory
      • Type - is the parent class.
      • UxmlTraits - is the next class.
      • All that is required it the contractor.
    • UXML Traits – Defines the UXML Traits for this element

    If you want to create your own elements you will need these 2 classes.
