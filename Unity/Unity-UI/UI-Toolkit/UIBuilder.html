UI Builder

Author: Matthew Williams

Date: 17/11/2021


Instructions

The new UI Builder introduces is a tool that will allow you to create UI for runtime as well as for your editor.
The UI Builder can be accessed via 2 different methods.
Either method 1.

• Click the window tab.

• Scroll down till you find the UI Toolkit.

• Open it and located the UI Builder button and click.
Or alternatively use method 2.
If you have an existing visual asset tree (.uxml), double click and it will open the UI Builder.
If you don’t have a visual asset tree (.uxml)

• In the project section right click in the assets folder or folder of your choice.

• Create, UI Toolkit, UI document and that will create the visual asset tree (.uxml) file.

When that is done open the UI Builder and I will show you a brief over view of the UI Builder layout as I will be going in to depth in subsequent videos in this series

Layout

The layout is split into 3 parts

Middle

1) the middle being the Viewport when you drag elements onto the canvas or add them to the hierarchy they will be displayed here

Viewport

a) the viewport contains several features

i) file: can open, new, save, save as

ii) zoom: by percentage

iii) fit canvas: will type to fit your canvas to the screen it will shrink it unless it is set to match game view.

iv) Theme: set a theme for the canvas

v) Preview: lets you preview how the canvas will work at when used of course functionality will not work.

vi) Settings: show UXML/ USS & Settings

vii) UXML/ USS: at the bottom will show you the code layout of the uxml & uss files.

Right

2) On the right side is the inspector when you have an element selected it will show 3 drop down tabs.

Type

a) the first one being type specific if you select a button, it will have button specific values and label will have label specific values, each element will have these values in common.

i) Name: you can name the element and it will become an #Name that can be referenced in the stylesheet.

ii) View data key: used for view data persistence (i.e., tree expanded states, scroll position and zoom level).

iii) Picking mode: determines if this element can be picked during mouse events or Ipanel.pick queries

iv) Tool tip: when hover it will display a tool tip that can be used

v) Using hints: offers a set of values that describe the intended usage patterns of a specific visual element it needs to be set before being add to a panel, as when it’s added it will effectively become read-only.

vi) Tab index: an integer used to sort focusable in the focus ring. Must be greater than or equal to zero.

vii) Focusable: a Boolean that if true will allow this element to be focused

Stylesheet

b) The second tab is the Stylesheet, it has a text box, 2 buttons and a drop-down tab.

i) When you press the first button called “Add style class to list” it will take the text from the text box and add the class by that name for example “Test” will add. Test to this element.

ii) When you press the second button called “Extract inline styles to new class” it will take the changed values from the 3rd tab which I will get in a bit and create a new class for them named after the text from the text box.

iii) And lastly the drop-down tag called “matching selectors” when an .Class, #Name or Type matches the element it will show a drop-down tab that contains all of the style it will add to the element unless it is overwritten by…

inlined styles

c) The last tab is “inlined styles” this tab contains a list of drop-down tabs that do various different changes to the element summarily to what CSS does to HTML as this new system take a lot of inspiration for its structure and style sheet.

i) Display controls everything to do with the visual of the element.

ii) Position controls as its name suggests controls its position on the canvas

iii) Flex is for group element controls and spacing

iv) Align how the elements will be aligned on the canvas

v) Size as its name would imply controls the sizes of the element min max and current size using either px, %

vi) The margin & padding works the same here as in CSS margins push other elements away from the current element and padding shrinks the element working space while maintaining its size

vii) Text is only used for elements such as the label that works with text

viii) Background you can set a color for the element add image and tint it to the background of the element

ix) Border like CSS will add borders around the element you can set a colour for each side its width and if the corners have a radius (curve)

x) Transform controls the pivot point on the element, translate an offset position relative to were its located, change its scale and rotate around the pivot point

xi) Curser will change the mouse curser to the selected image when it is hovered over.

xii) Transition Animations allows you to create animation to the element.

Left

3) On the left side is the “Stylesheet”, “Hierarchy”, “Library”

Stylesheet


a) Stylesheet will allow you too create or import an existing style sheet (.uss)

i) In the text box label “Add new selector” it will replace the text with a “.” And open a popup

ii) The popup will it will layout how to write your selector

iii) When you create a selector, you can click it an it will create 2 tabs in the inspector on the right

iv) The first tab is style selector and it contains the name of the selector and can be renamed here

v) The last tab contains the same tabs as the “inlined styles” and will affect all element that implement this selector

Hierarchy


b) Hierarchy is the structure of starting off with the root ({name}.uxml)

i) You can drag & drop elements from the library here to add them to the canvas

ii) and change its location in relation to the tree as a whole

iii) you have option when right clicking an element one of which is template it will take the element and its children and make a Visual tree asset (.uxml) file that contains the element and its children and can be located in the project button in the library.

library


c) In the library tab it starts off with 2 buttons “Standard” & “Project” each containing tabs

standard

i) With the standard button selected it shows 2 normal tabs and 5 editor only tabs that can be enabled by clicking on the root element and selecting the “editor extension authoring” Boolean in the inspector or in the Edit, Project Settings, UI Builder – “enable editor extension authoring by default” if this one is selected you will need to disable it in the UI Builder every time the editor only tabs can’t be used in you game build

ii) As it stands the tabs are Containers, Controls for the normal tabs & the editor only tabs are Numeric Fields, Value Fields Choice Fields, Toolbar, inspectors

iii) Containers contains

(1) Visual element: this is the most basic of containers and is the base of all elements

(2) Scroll view: which functions the same as the legacy UI system.

(3) List View: which functions the same as the legacy UI system.

(4) IMGUI Container is the same as visual element but has focusable true

(5) Group box does the same as visual element but has 2 more variables binding path & text

(a) Bonding path is the path of the target property to be bound

(b) Text is just text and is affected but the “Text” option tab and effects it’s children if unset

iv) Controls these are the same as the legacy UI system

(1) Label, button, toggle, scroller, text field, foldout, slider, slider(int), min max slider, progress bar dropdown radio button & group

v) Numeric fields for the editor only

(1) Integer, float, long, progress bar, vector2, vector 3, vector 4, rect, bounds, vector2(int), vector3(int), rect(int), bounds(int), object field

vi) Value field for the editor only

(1) Color, curve, gradient

vii) Choice field for editor only

(1) Enum, tag, mask, layer, layer mask

viii) Toolbar

(1) Toolbar

(2) Menu

(3) Button

(4) Spacer

(5) Toggle

(6) Breadcrumbs

(7) Search field

(8) Popup search field

ix) Inspectors

(1) Property field

Project


(1) The project button shows all .UXML located inside of your project and some of these can be templates you have created for your project.
