Runtime UI

First you need to right click the Hierarchy -> UI Toolkit -> UI Document
When you create the Game object it comes with a Component called UI Document it has 3 exposed variables
Called Panel Settings, Source Asset & Sort Order.
If you created it though this method a Folder is added in your Assets called “UI Toolkit” it contains
A panel Settings asset by the same name and a folder with a theme style sheet
If the panel Settings is empty apply the panel settings file.
Panel Settings the panel settings asset to be used to display the UI in its panel.
The panel is what is rendered to the screen the Source Assets are added to the panel based on the sorting order and if it is a child object of another UI Document

The Source Asset that requires a Visual Tree Asset is you’re (.UXML) file

Editor Window

First you need to right click the Assets -> Create -> UI Toolkit -> Editor Window
Then type the text into it you can choice if you want the uxml or uss but for the C# file is needed for the editor window unless you already have one and are going to attach a new uxml file to your existing C# file.

When it is created the window will pop up and it will have 3 labels created 2 from the script (one with the uss attached) and the last 1 from the uxml document.
This is the example they give you to teach the new editor windows style
Open the C# script and you will see:
It has using unity engine & editor UIElements as this is what UI Toolkit was originally called
3 functions the first is Show Example and has a menu item attribute and show where you can access the editor window
And the function is the same as the original editor window

Next is the Create GUI which is called when the window is opened

the first line is the root element and it serves the same function as the panel settings.
The next block of text is the first text label from the editor window that is added to the root element.
Then comes the UXML that is imported from the asset and is in Instantiated and added to the root
And the last block of code is the uss style sheet imported then a label is created the same way as the first but the stylesheet is added to the label which is added to the root

As the editor window shows how the order, they are added to the root informs you that they will be loaded in 1 by 1.

Editor

First you need a C# script to edit for this example I will uses a basic player script with current/max health and level
Next you need a folder called “Editor” so when you build your game it’s not included in the build as it may break your build.
Then create a C# script with the suffix of “Editor” as it is a common practice so as to make sure you know it is an editor script
You will need to add using UnityEngine.UIElements; & using UnityEditor.UIElements;
Add the attribute [CustomEditor(typeof(*ScriptName*))] above the class then replace monobehaviour with Editor
I have 2 functions here in my script called “OnInspectorGUI” & “CreateInspectorGUI” the first is the old function that allows for the editing of the script’s component in the inspector using the old method and the second is for new visual element it differs slightly from the editor window as it doesn’t have a “rootvisualelement” that you add to, for the editor function it returns a visual element and that visual element will serve as the root element that you will add to.
You can create your UI in the code here and add to your root visual element or you can create it in the UI Builder and add that to the root as I have done here.
